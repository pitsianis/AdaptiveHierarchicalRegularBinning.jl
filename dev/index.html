struct CountSortDetails{B, D}

Contains all of the details needed for the countsort_xxx_impl! algorithms.

Fields

  • D: The leading dimension.

AdaptiveHierarchicalRegularBinning.RadixSortDetails — Type
struct RadixSortDetails{B, D}

Contains all of the details needed for the radixsort_xxx_xxx_impl! algorithms.

Fields

  • B: The length of the bit mask to use.

  • D: The leading dimension.

  • csd: The underlying CountSortDetails object.

  • par_th: Threshold for parallel-to-parallel execution.

  • seq_th: Threshold for parallel-to-sequential execution.

  • sml_th: Threshold for skiping small arrays.

  • dpt_th: recursion threshold.

AdaptiveHierarchicalRegularBinning.bit_interleave — Method
bit_interleave([M, ]W)

Interleaves an array of unsigned integers.

Arguments

  • M: The method with which to interleave
  • W: The array

Examples

julia> bit_interleave([0x00FF, 0x000F]) |> bitstring
"0101010111111111"

julia> bit_interleave([0x000F, 0x00FF]) |> bitstring
"1010101011111111"

julia> bit_interleave([0x0080, 0x0001]) |> bitstring
"0100000000000010"

julia> bit_interleave([0x0001, 0x0080]) |> bitstring
"1000000000000001"

AdaptiveHierarchicalRegularBinning.bit_space — Method

bit_space(M, w, n)

Evenly space out bits.

Arguments

  • M: The method to space out the bits
  • w: The word to space out
  • n: The amount of padding

Examples

julia> bit_space(Brute, 0x00ABCDEF, 0) |> bitstring
"00000000101010111100110111101111"

julia> bit_space(Brute, 0x00ABCDEF, 1) |> bitstring
"10100000101000101010100010101010"

julia> bit_space(Brute, 0x00ABCDEF, 2) |> bitstring
"00000100100100100000100100100100"

julia> bit_space(Brute, 0x00ABCDEF, 3) |> bitstring
"10001000100000001000100010001000"

julia> bit_space(Pdep, 0x00ABCDEF, 2) == bit_space(Brute, 0x00ABCDEF, 2)
true

AdaptiveHierarchicalRegularBinning.countsort_par_impl! — Method
countsort_par_impl!(Va, Ra, Ia, Vb, Rb, Ib, C, csd)

Parallel countsort.

Remarks

Inputs Va, Ra, Ia are not altered.

Arguments

  • Va: The cloud of points.

  • Ra: The morton transformation for each point of Va

  • Ia: The current permutation of Va.

  • Vb: A matrix to store the sorted version of Va.

  • Rb: A vector to store the sorted version of Ra.

  • Ib: A vector to store the resulting permutation.

  • C: A matrix acting as a working space for the countsort algorithm.

  • csd: The CountSortDetails object used to configure the algorithm.

radixsort_par_par_impl!(Va, Ra, Ia, Vb, Rb, Ib, P, rsd, allocator)

Parallel-to-Parallel radixsort.

Arguments

  • Va: The cloud of points.

  • Ra: The morton transformation for each point of Va

  • Ia: The current permutation of Va.

  • Vb: An auxiliary matrix to store the partially sorted version of Va.

  • Rb: An auxiliary vector to store the partially sorted version of Ra.

  • Ib: An auxiliary vector to store the resulting permutation.

  • P: A ping-pong matrix that denotes where the latest results are.
